# AGENTS.md

## プロジェクト概要
このリポジトリは、佐藤医院の公式Webサイトをリニューアルするための制作案件です。
目的は、患者様が必要な情報へ素早くたどり着けること、清潔感と安心感のある印象を伝えること、スマートフォンでも見やすく使いやすいサイトにすることです。

## このファイルの扱い
- `requirements/` フォルダ内の制作依頼書・掲載原稿は、このプロジェクトの正式な資料とすること。
- この `AGENTS.md` は、それらの正式資料を要約し、作業上の判断基準を補足するルールブックとして扱うこと。
- 実作業では、この `AGENTS.md` を起点にしつつ、正式資料の内容と整合することを必ず確認すること。
- 制作依頼書や掲載原稿が更新された場合は、それに合わせてこの `AGENTS.md` も更新する運用とすること。

## 対象ファイル
主に次のファイルを編集対象とします。
- `index.html`
- `about.html`
- `medical.html`
- `news.html`
- `faq.html`
- `css/style.css`
- `js/script.js`
- `assets/`

## 変更方針
- 依頼された範囲を最優先し、不要な独自判断は避けること。
- 既存の掲載文言は勝手に書き換えないこと。
- 文章の言い換え、要約、追記は、明確な指示がある場合のみ行うこと。
- 画像やロゴは、基本的に既存の支給素材を使うこと。
- 外部素材や新規素材を使う必要がある場合は、事前に影響を確認すること。
- 変更は、既存の導線やデザインを壊さない範囲で行うこと。

## 機能要件
- レスポンシブ対応を必ず維持すること。
- お知らせはトップページに最大5件表示すること。
- Googleマップはトップページと診療時間・所在地ページの両方に設置すること。
- アクセス情報には、駐車場案内と公共交通機関の利用方法を掲載すること。
- ご予約・お問い合わせページは、必要に応じて提案してよい。

## デザイン要件
- 白を基調に、淡いグリーンまたはブルーをアクセントカラーとして使うこと。
- 清潔感、親しみやすさ、地域密着の印象を重視すること。
- 写真を多く使い、視覚的に分かりやすくすること。
- 高齢者にも読みやすいフォントサイズと行間を確保すること。
- キャッチコピー「地域に根ざした医療」を目立たせること。
- 必要以上に派手な演出は避けること。

## トップページの構成
トップページは、次の流れを意識して構成すること。
1. ファーストビュー
2. 診療時間
3. お知らせ
4. 院長挨拶
5. 診療内容
6. 当院について
7. スタッフ紹介
8. アクセス
9. フッター

## コンテンツルール
- 掲載内容は、医療機関サイトとして読みやすさ、信頼感、誤解のなさを最優先すること。
- 診療時間、所在地、連絡先などの基本情報は、ページ間で矛盾しないようにすること。
- アクセス情報は、徒歩、自転車、車、公共交通機関の導線が分かるように整理すること。
- 写真は、内容理解を助ける箇所に優先的に配置すること。

## コーディングルール
- HTML は意味の通る構造を優先し、見出し階層やランドマークを崩さないこと。
- CSS は保守しやすい書き方を心がけ、色、余白、フォントの指定に一貫性を持たせること。
- JavaScript は必要最小限にとどめ、表示や操作に不要な複雑化を避けること。
- 追加するコードは、既存のデザインや導線を壊さない範囲で実装すること。

## レスポンシブ対応
- スマートフォン、タブレット、PC の各幅で崩れないことを必須とする。
- 文字サイズ、行間、余白、表組み、画像比率が小画面でも読みやすいことを確認する。
- 横スクロールが発生しやすい要素は、折り返しや幅調整を行うこと。
- モバイルでもナビゲーションが迷わず操作できることを重視すること。
- 高齢の利用者を想定し、タップ領域と可読性を十分に確保すること。

## 修正時のルール
- 既存の掲載文言を勝手に書き換えないこと。
- 画像、ロゴ、外部素材の差し替えは、要件と支給素材の範囲を確認してから行うこと。
- 変更が必要な場合は、どの要件に基づく修正かを明確にしてから実施すること。
- 影響範囲が広い修正は、先に全体構成への影響を確認してから進めること。
- デザイン調整は、清潔感、親しみやすさ、地域密着の印象を損なわない範囲で行うこと。
- 既存の表示順や情報整理の方針を変える場合は、要件との整合を必ず確認すること。

## 完了時の確認事項
- 依頼内容とこの `AGENTS.md` の要件を反映できていること。
- 掲載文言が原稿どおりで、改変や抜け漏れがないこと。
- トップページの構成順が意図どおりで、アクセスとフッターが最後にあること。
- お知らせ、診療内容、Googleマップ、アクセス情報の要件を満たしていること。
- スマートフォン、タブレット、PC での表示崩れがないこと。
- 支給素材以外を使用していないこと。
- 変更対象以外のファイルを更新していないこと。
