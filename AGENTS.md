# AGENTS.md

## プロジェクト概要
本プロジェクトは、佐藤医院の公式Webサイトをリニューアルするための制作案件です。
目的は、患者様が必要な情報へ素早くたどり着けること、清潔感と安心感のある印象を伝えること、スマートフォンでも見やすく使いやすいサイトにすることです。

## 最初に参照する資料
作業を始める前に、必ず次の順で確認してください。
1. `requirements/制作依頼書.html`
2. `requirements/掲載原稿.html`

特に `掲載原稿.html` は掲載テキストの正本です。文言を追加・変更する前に、必ず依頼内容と整合するか確認してください。

## 制作ルール
- 依頼書の要件を最優先し、記載のない独自判断は避けること。
- `掲載原稿.html` の本文は原則改変しないこと。表現の言い換え、要約、追記は行わない。
- 画像は支給素材を基本とし、支給画像以外を使う場合は事前確認を前提とすること。
- ロゴは `logo.png` を使用すること。必要に応じて別案は検討可だが、勝手に差し替えないこと。
- トップページは、ファーストビュー、診療時間、お知らせ、院長挨拶、診療内容、当院について、スタッフ紹介、アクセス、フッターの流れを意識して構成すること。
- アクセス情報とフッターは必ず最後に配置すること。
- お知らせはトップページで最大5件、診療内容はトップページで最大4件を基本とすること。
- Googleマップはトップページと診療時間・所在地ページの両方に設置すること。
- ご予約・お問い合わせページは提案可だが、追加する場合は既存要件との整合を保つこと。

## コーディングルール
- HTML は意味の通る構造を優先し、見出し階層やランドマークを崩さないこと。
- CSS は保守しやすい書き方を心がけ、色・余白・フォントの指定は一貫性を保つこと。
- JavaScript は必要最小限とし、表示や操作に不要な複雑化を避けること。
- 文言、見出し、診療時間、所在地、連絡先などの基本情報は `掲載原稿.html` と矛盾させないこと。
- 医療機関サイトとして、読みやすさ、信頼感、誤解のない表現を優先すること。
- 追加するコードは、既存のデザインや導線を壊さない範囲で実装すること。

## レスポンシブ対応
- スマートフォン、タブレット、PC の各幅で崩れないことを必須とする。
- 文字サイズ、行間、余白、表組み、画像の比率が小画面でも読みやすいことを確認する。
- 横スクロールが発生しやすい要素は、折り返しや幅調整を行うこと。
- ナビゲーションはモバイルでも迷わず操作できることを重視すること。
- 高齢の利用者も想定し、タップ領域と可読性を十分に確保すること。

## 修正時のルール
- 既存の掲載文言を勝手に書き換えないこと。
- 画像、ロゴ、外部素材の差し替えは、要件と支給素材の範囲を確認してから行うこと。
- 変更が必要な場合は、どの要件に基づく修正かを明確にしてから実施すること。
- 影響範囲が広い修正は、先に全体構成への影響を確認してから進めること。
- デザインの調整は、清潔感・親しみやすさ・地域密着の印象を損なわない範囲で行うこと。
- 既存の表示順や情報整理の方針を変える場合は、要件との整合を必ず確認すること。

## 作業完了時の確認事項
- `制作依頼書.html` と `掲載原稿.html` の要件を反映できていること。
- 掲載文言が原稿どおりで、改変や抜け漏れがないこと。
- トップページの構成順と、アクセス・フッターが最後にあることを確認すること。
- お知らせ、診療内容、Googleマップ、アクセス情報の要件を満たしていることを確認すること。
- スマートフォン、タブレット、PC での表示崩れがないことを確認すること。
- 支給素材以外を使用していないことを確認すること。
- 変更対象以外のファイルを更新していないことを確認すること。
